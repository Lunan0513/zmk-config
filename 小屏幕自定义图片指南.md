# Silakka54 + Nice!View 小屏幕自定义图片指南

## 硬件环境
- **键盘**: Silakka54 分体键盘
- **控制器**: nice!nano v2 (nRF52840)
- **屏幕**: Nice!View (Sharp LS011B7DH03 内存屏, 160×68 像素, 单色)
- **固件**: ZMK v0.3

## 仓库结构
- **zmk-config**: `https://github.com/Lunan0513/zmk-config` — 键盘配置仓库
- **zmk-nice-oled (fork)**: `https://github.com/Lunan0513/zmk-nice-oled` — 屏幕模块 fork（用于存放自定义图片）

## 当前方案（最终可工作的版本）

使用成熟开源模块 **zmk-nice-oled**（fork 版本），在 `custom_status_screen.c` 中替换为自定义兔子图片。

### 文件关系

```
zmk-config/
├── .github/workflows/build.yml    → 引用 ZMK v0.3 官方 CI
├── build.yaml                     → 定义 shield 组合
└── config/
    ├── west.yml                   → 引用 ZMK + zmk-nice-oled 模块
    ├── lily58.conf                → 键盘配置
    └── lily58.keymap              → 键位布局

zmk-nice-oled-fork/
└── boards/shields/nice_oled/
    ├── assets/rabbit_images.c     → 🐰 自定义图片 C 代码
    └── custom_status_screen.c     → 显示兔子图的入口
```

### build.yaml（关键配置）

```yaml
---
include:
  - board: nice_nano_v2
    shield: lily58_left nice_view_adapter nice_epaper
    snippet: studio-rpc-usb-uart
    cmake-args: -DCONFIG_ZMK_STUDIO=y -DCONFIG_ZMK_STUDIO_LOCKING=n
  - board: nice_nano_v2
    shield: lily58_right nice_view_adapter nice_epaper
  - board: nice_nano_v2
    shield: settings_reset
```

**关键点**：
- `nice_view_adapter` 提供 SPI 引脚配置 → 必须有
- `nice_epaper` 提供显示设备定义 + LVGL 驱动 → 必须有
- **不需要** `nice_view`（它是另一个 shield，不需要）
- **不需要** 自定义 shield（不需要 `boards/shields/my_rabbit`）
- **不需要** `zephyr/module.yml`（zmk-nice-oled 自己带了）

## 图片格式规范

| 参数 | 值 |
|------|-----|
| 格式 | 1-bit 黑白 PNG |
| 尺寸 | **160 × 68 像素** |
| 转码格式 | LVGL `LV_IMG_CF_INDEXED_1BIT` |
| 调色板 | 8 字节，支持 `CONFIG_NICE_OLED_WIDGET_INVERTED` 切换 |
| 像素数据 | 1 bit/pixel，逐行排列，每行 20 字节 |
| 左手图 | 文件名 `Left.png`，竖版 68×160 → 代码自动旋转 90° + 垂直翻转 |
| 右手图 | 文件名 `right.png`，横版 160×68 → 代码自动垂直翻转 |
| 像素映射 | 亮色/透明 → `1`（黑），暗色 → `0`（白）|

## 更新图片流程

### 第 1 步：编辑图片
在 `图片/` 目录下编辑 `Left.png` 和 `right.png`：
- 160×68 像素（左手图竖版 68×160 也可以，代码自动旋转）
- 黑白（可含透明度，透明 = 白色）

### 第 2 步：转码 → C 数组
用 Python 脚本把 PNG 转成 LVGL C 代码，输出到 fork 仓库：
```
zmk-nice-oled-fork/boards/shields/nice_oled/assets/rabbit_images.c
```

转码脚本会自动：
- PNG 反滤镜（处理 Sub/Up/Average/Paeth 滤镜）
- 90° 旋转（如果宽 < 高）
- 垂直翻转
- 颜色反转（亮→黑）
- 生成 `LV_IMG_CF_INDEXED_1BIT` 格式

### 第 3 步：Push fork
```bash
cd zmk-nice-oled-fork
git add boards/shields/nice_oled/assets/rabbit_images.c
git commit -m "update rabbit images"
git push --force
```

### 第 4 步：编译
在 zmk-config 仓库触发 Actions：
```bash
cd zmk-config
# 任意修改触发 push，或去 Actions 页面手动 Run workflow
git commit --allow-empty -m "trigger build" && git push
```

### 第 5 步：刷入
1. 下载 Actions Artifacts 压缩包
2. 左手 nice!nano **双击 RST** → 弹出 `NICENANO` U 盘 → 拖入 `*_left-*.uf2`
3. 右手 nice!nano **双击 RST** → 弹出 `NICENANO` → 拖入 `*_right-*.uf2`

## 踩过的全部坑（按时间顺序）

| # | 错误现象 | 根因 | 正确做法 |
|---|---------|------|---------|
| 1 | shield 找不到 | `my_rabbit` 放在废弃路径 `config/boards/shields/` | ZMK v0.3 废弃了此路径 |
| 2 | 自定义 shield 注册失败 | 缺少 `Kconfig.shield` | 每个 shield 必须有 `Kconfig.shield` |
| 3 | CMake 配置错误 | 缺少 `zephyr/module.yml` | 仓库根目录需要 `zephyr/module.yml` |
| 4 | Kconfig 重复配置警告 | `lily58.conf` 里 `ZMK_DISPLAY` 写了两次 | 不要重复 |
| 5 | choice 符号 default 无效 | `Kconfig.defconfig` 对 `ZMK_DISPLAY_STATUS_SCREEN_CUSTOM` 直接设 default | choice 符号必须通过 `choice` 块设置 |
| 6 | 右手侧 USB 冲突 | `ZMK_STUDIO` 写在共享 `.conf` 里 | 只通过 `cmake-args` 对左手启用 |
| 7 | 链接错误 `lv_img_set_src` 未定义 | 没有启用 `LV_USE_IMG` | 在 `.conf` 加 `CONFIG_LV_USE_IMG=y` |
| 8 | 链接错误 `zmk_split_get_role` 未定义 | 头文件包含有问题 | 不能运行时判断左右 |
| 9 | `fatal error: zmk/display.h` | `CMakeLists.txt` 缺少 include 路径 | 加 `zephyr_library_include_directories(...)` |
| 10 | **`my_rabbit.c` 没被编译进固件** | **没有 `CMakeLists.txt`** | **每个自定义 shield 必须有 `CMakeLists.txt`** |
| 11 | 屏幕全黑不显示 | `nice_view_adapter` 只有 SPI 引脚，没有显示设备 | 必须配合 `nice_epaper` 或 `nice_view` |
| 12 | 图片撕裂纹 | PNG 解压后没做反滤镜（unfilter）| 必须按 PNG 规范处理 Sub/Up/Average/Paeth |
| 13 | 图片朝向错误 | 竖版图没旋转，所有图没做垂直翻转 | 代码自动处理旋转+翻转 |
| 14 | 颜色黑白反转 | 像素映射方向反了 | 亮色→黑(1)，暗色→白(0)|

## 核心教训

> **不要自己写 shield 处理 Nice!View。** 直接用 `zmk-nice-oled` 模块的 `nice_epaper` shield，它已经正确处理了：
> - Sharp LS0xx 显示设备定义（overlay）
> - LVGL 配置（VDB、DPI、color depth、dedicated queue）
> - Kconfig choice 项的正确设置方式
> - 所有 LVGL 子模块的启用（img、canvas、animimg、animation、label）
> - 字体加载
> - 编译依赖（CMakeLists.txt + include 路径）

> **fork 它的仓库，只改图片文件和 `custom_status_screen.c`**，其他不动。

> **转码工具必须正确处理 PNG 滤镜。** 不可以用简单的 `zlib.decompress()` 直接取像素值，必须按 PNG spec 对每行做 unfilter。

> **Nice!View 是 160×68 横屏安装在竖板上的。** 图片在屏幕上的方向要实际测试确认。
